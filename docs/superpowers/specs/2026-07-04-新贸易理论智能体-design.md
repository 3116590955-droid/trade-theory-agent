# 新贸易理论智能体 · 设计规格文档

**日期：** 2026-07-04
**状态：** 待实施

---

## 背景与目标

为国际经济学课程的同学构建一个可共享的 AI 智能体，专注于新贸易理论（New Trade Theory）领域。智能体需同时处理有标准答案的概念/模型题，以及需要多角度论证的开放性分析题，所有回答必须基于可检索的课程材料或学术文献。

---

## 整体架构

```
用户提问
    ↓
System Prompt（角色定位 + 回答规则）
    ↓
问题分类判断
    ├─ 标准题 → Filesystem MCP 检索本地知识库 → 结构化三层回答
    └─ 开放性题 → Filesystem MCP + arXiv MCP → 多角度有据论证
    ↓
结构化输出（定义 → 直觉 → 模型 → 区别 → 案例）
```

### MCP 分工

| MCP | 用途 | 优先级 |
|---|---|---|
| Filesystem MCP | 读取本地课程材料（PDF/PPT/讲义） | 主力，优先使用 |
| arXiv MCP | 检索前沿学术文献 | 补充，本地不足时使用 |
| Web Search MCP | 处理超出知识边界的话题 | 兜底，100 token 内简短总结 |

---

## System Prompt 设计

完整内容见 `system_prompt.md`，核心模块：

1. **身份与知识边界**：Krugman 1979/1980、Helpman-Krugman、Eaton-Kortum 2002、Melitz 2003、战略性贸易政策
2. **知识来源规则**：先 Filesystem → 后 arXiv → 两者不足则明确说明
3. **问题分类逻辑**：标准题（结构化五点）/ 开放性题（多角度论证）
4. **输出格式**：三层结构（Level 1 直觉 / Level 2 模型 / Level 3 严格分析）
5. **边界规则**：不追问、不超出知识边界、不编造文献

## 边界处理规则

- 实时贸易数据查询：告知用户使用 WTO/UN Comtrade
- 超出新贸易理论范围的话题（如宏观货币政策、微观企业管理）：使用 Web Search MCP 检索，100 token 以内简短总结，并注明"此内容超出核心知识库范围"

## 部署方式

目标为网页端部署，供多位同学共享访问。

**推荐方案：** 基于 Claude API 构建轻量网页应用
- 前端：简单的对话界面（HTML/JS 或 Next.js）
- 后端：调用 Claude API，system prompt 固定注入
- 知识库：将本地 PDF/PPT 预先转为文本，存储为向量数据库或 Markdown 文件随请求检索
- arXiv：通过 API 集成，按需调用

本地 Claude Desktop 方案仍可作为开发调试阶段使用。
